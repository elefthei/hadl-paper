\section{Formal Model}
\label{sec:semantics}

This section gives an operational semantics for \pact that alternates evaluation, type checking, and policy checking, with self-healing when any check fails. Types, workflows, and authorization policies materialize at runtime and reduction deliberately visits ill-typed intermediate states during self-heal, so we (i) parameterize over a non-deterministic oracle, (ii) use substitution-based reduction so that materialized values flow by $\beta$-reduction rather than through a typed environment, and (iii) state soundness relative to an oracle that is eventually-truthful within the declared retry budget.

\subsection{Machine Configuration and Judgments}
\label{sec:sem-config}

A \pact configuration is a four-tuple $C = \MConfigS{\MErr}{\MPol}{\MStore}{e}$ where $\MErr$ is a \emph{heal context} holding only failed-attempt feedback events $\ErrEv{\epsilon}$ --- flushed to $\emptyset$ on a successful oracle step and extended by $\cdot \ErrEv{\epsilon}$ on a heal step --- whose length $|\MErr|$ is the retry count; $\MPol$ is the installed Cedar policy, updated only by $\Enforce{v}$ via an append-only merge that only tightens; $\MStore$ is a finite map from mutable-variable names to $(\tau, v)$ pairs, mutated only by \texttt{var} declarations and assignments; and $e$ is the expression under reduction. The store $\MStore$ is orthogonal to the substitution-based core: \texttt{let}, closure application, and $\rulename{For-Cons}$ use capture-avoiding substitution, while \texttt{var} declarations, assignments, and reads use $\MStore$; the two surfaces do not interact. Rules that leave $\MStore$ unchanged elide it and use the three-component notation $\MConfig{\MErr}{\MPol}{e}$; rules that read or write it are collected in \sect{app:rules-mut}. The current principal lives inside the $\Gen{\tau}{s}{\MPrin}$, $\Agent{s}{\MPrin}$, and closure-application syntax. Values are the subset satisfying $\mathsf{isValue}$ (base literals, records, arrays, schema values $\Tschema(\tau)$, Cedar policy values, closures).

Four judgments drive correctness. \emph{Runtime typing} $\StType{v}{\tau}$ holds of closed values (substitution keeps every checked value closed). \emph{Residual static typing} is black-boxed: it is re-invoked at runtime after every materialization exactly as at compile time; only the reflexivity cases the soundness proof uses are exposed. \emph{Principal membership} $\MPrin \in \mathrm{dom}(\MPol)$ checks at runtime whether the calling principal is registered in the installed policy; a mismatch ($\MPrin \notin \mathrm{dom}(\MPol)$) triggers self-heal via $\rulename{Gen-PolHeal}$/$\rulename{Agent-PolHeal}$. \emph{Well-formedness} $\WF{C}$ requires both $|\MErr| \le \MRetry$ and that every store cell is well-typed: $\forall x,\tau,v.\; \MStore(x) = (\tau, v) \Rightarrow \StType{v}{\tau}$. Runtime typeability lives inside the reduction rules as $\StType{\cdot}{\cdot}$ side conditions that substitution composes.

\subsection{Reduction}
\label{sec:sem-reduction}

The reduction relation $C \step C'$ is a single small-step relation parameterized by the oracle $\mathcal{O}$, split between \emph{root-level} rules that fire when the top of $e$ matches and \emph{structural congruence} rules that drive reduction to the leftmost sub-term under CBV. Let-binding is $\beta$-reduction: $\rulename{Let-Bind}$ rewrites $\Let{\tau}{v}{e}$ to $\Subst{e}{v}$, the capture-avoiding substitution of $v$ for $x$ in $e$; closure application $\rulename{Eval-Success}$ performs the analogous simultaneous substitution.

Self-healing is \emph{let-centric} and \emph{continuation-driven}. The expression $\Gen{\tau}{s}{\MPrin}$ is no longer a standalone redex: it can only reduce as the immediate right-hand side of a $\texttt{let}$. We classify materialization targets by the predicate $\Healable{\tau}$ --- true on Schema, Policy, arrow types, and any record/array type containing a healable component; false on the base scalars Unit, Bool, Number, String. At each healable $\tau$ the runtime re-checks the oracle's response \emph{and} re-typechecks the continuation $p$ under the singleton context $\Gext{x}{\tau}$; the second of those checks is what drives self-heal. Value checks use the judgment $\StType{v}{\tau}$ (closed values, empty context); continuation checks use $\TJok{\Gext{x}{\tau}}{p}{\tau'}$ with $r \in \{\mathsf{ok}, \mathsf{err}(\epsilon)\}$, where the error branch returns the feedback string $\epsilon$ consumed by self-healing (Appendix~\ref{app:rules} lists all rules).

Each healable $\tau$ has exactly two rules: $\rulename{Gen-Ok}_\tau$ fires when both $v$ and the continuation type-check, stepping to $\Subst{p}{v}$ and flushing $\MErr$; $\rulename{Gen-Heal}_\tau$ fires when only $v$ does, extending $\MErr$ with the continuation's $\epsilon$ and retrying. Two uniform rules apply at every $\tau$: $\rulename{Gen-TypeErr}$ steps an ill-typed response at non-healable $\tau$ to the fail-closed sink $\errV$ (intentionally untypeable: $\TJok{\emptyset}{\errV}{\tau}$ is unprovable for every $\tau$), and $\rulename{Gen-PolHeal}$ records a feedback event and retries when $\MPrin \notin \mathrm{dom}(\MPol)$ (principal mismatch). \figr{fig:rules-core} shows the Schema pair; the arrow pair is structurally identical with $\tau = \Tarrow{T_1 \to \cdots \to T_n}{T'}$ and is given in Appendix~\ref{app:rules}. For arrows, $\Subst{p}{v}$ substitutes the oracle-returned closure into the continuation, mechanized via the \texttt{lean-subst} framework. The full rule set --- including the $\rulename{Gen-Budget}$ sink and the Policy pair, which is subsumed by the parametric $\rulename{Healable-Ok}$/$\rulename{Healable-Heal}$ rules via $\Healable{\Tpolicy} = \mathsf{true}$ --- appears in Appendix~\ref{app:rules}.

Because $\errV$ has no runtime type, T2 (staged materialization, \sect{thm:pact-soundness}) is stated as ``$v \neq \errV \Rightarrow \exists \tau.\ \StType{v}{\tau}$''. The judgment $\StType{v}{\tau}$ checks a closed value against type $\tau$ and is used wherever the feedback string is irrelevant (e.g., T2 and $\rulename{Let-Bind}$). The standalone $\Agent{s}{\MPrin}$ keeps its own pair $\rulename{Agent-Success}$/$\rulename{Agent-PolHeal}$: agent's materialization type $\Tstring$ is non-healable, so no type-heal rule is needed; conceptually $\Agent{s}{\MPrin}$ is shorthand for $\Let{\Tstring}{\Agent{s}{\MPrin}}{\unit}$.

\subsection{Correctness}
\label{sec:sem-correctness}

Classical progress and preservation do not apply directly: types are runtime values, policies evolve, and heal steps deliberately visit ill-typed configurations. We decompose what \emph{is} preserved into three per-step \emph{Soundness} components plus \emph{Progress} for oracle redexes, all parameterized by the oracle, and argue \emph{Termination} on paper.

\begin{definition}[Eventually-truthful oracle]
\label{def:oracle-truthful}
An oracle $\mathcal{O}$ is \emph{eventually-truthful at budget $\MRetry$} for a site $(s,\tau)$ under an authorization predicate $A : V \to \mathrm{Prop}$ if there exists a heal context $\MErr$ with $|\MErr| \le \MRetry$ and a value $v$ satisfying $\Oracle{s}{\MErr}{\tau} \ni v$, $\StType{v}{\tau}$, and $A(v)$. We instantiate $A(v) \equiv \top$ in the soundness theorems below; $A$ lets callers (e.g., agent sites) rule out self-contradictory responses without weakening the progress lemma.
\end{definition}

\begin{sloppypar}
This matches the existential shape of the Lean predicate $\mathtt{Oracle.eventuallyTruthful}$: it witnesses some well-typed response at \emph{some} heal context, not the caller's current $\MErr$.
\end{sloppypar}

\begin{theorem}[Soundness]
\label{thm:pact-soundness}
For any oracle $\mathcal{O}$ and any step $C \step C'$:
(\textbf{T1}, WF-preservation) $\WF{C} \Rightarrow \WF{C'}$; (\textbf{T2}, staged materialization) if $C' = \MConfigS{\MErr'}{\MPol'}{\MStore'}{v}$ with $\mathsf{isValue}(v)$ and $v \neq \errV$, then $\exists \tau.\ \StType{v}{\tau}$; (\textbf{T3}, policy monotonicity) $\Allow{\MPol'} \subseteq \Allow{\MPol}$.
\end{theorem}

T2 is conditional on $v \neq \errV$ because $\errV$ is the fail-closed sink produced by $\rulename{Gen-TypeErr}$ and $\rulename{Gen-Budget}$ (Appendix~\ref{app:soundness} expands this).

\begin{theorem}[Progress for oracle redexes]
\label{thm:pact-progress}
{\sloppy Let $C = \MConfigS{\MErr}{\MPol}{\MStore}{e}$ with $\WF{C}$. \textbf{T4a} (no heal past budget for standalone gen): if $e = \Gen{\tau}{s}{\MPrin}$ and $|\MErr| > \MRetry$, then $C$ has no successor at all (standalone $\Gen{\tau}{s}{\MPrin}$ is not a redex). \textbf{T4b} (truthful success at Schema): if $e = \Let{\Tschema}{\Gen{\Tschema}{s}{\MPrin}}{x}$, $\MPrin \in \mathrm{dom}(\MPol)$, and $\mathcal{O}$ is eventually-truthful at $(s,\Tschema)$, then $\exists C'.\ C \step C'$ via $\rulename{Schema-Ok}$. \textbf{T4c} (progress-gen-Schema, progress-agent): with the same principal-membership hypothesis and a direct oracle witness at the caller's $\MErr$, $\exists C'.\ C \step C'$.\par}
\end{theorem}

T4b/T4c cover Schema, arrow, and Policy targets via 1-line corollaries of the parametric Healable lemma; commentary and the trace-level termination argument are in Appendix~\ref{app:soundness} and \ref{app:term}.

\begin{sloppypar}
\noindent\textbf{Scope.} T1--T4c are mechanized in Lean~4 (\texttt{mechanization/Pact/}), sorry-free and depending only on the standard meta-axioms $\mathtt{propext}$, $\mathtt{Classical.choice}$, $\mathtt{Quot.sound}$ (build-time guard: \texttt{AxiomCheck.lean}). Trace-level termination is prose-only. Progress is conditional on an eventually-truthful oracle; absent that, the configuration fails closed by absence of transition. The threat-model gaps of \sect{sec:design-threat} are outside the scope of soundness; practical claims are the subject of \sect{sec:evaluation}.
\end{sloppypar}
