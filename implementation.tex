\section{Implementation}
\label{sec:implementation}

\hadl is implemented in Rust: PEG parser, type checker, and async tree-walking interpreter with no code generation. Each primitive delegates to a dedicated backend. \texttt{js} expressions and backtick interpolation run on an embedded JavaScript engine (Boa) with \hadl variables mirrored as globals; \texttt{gen} and \texttt{agent} calls are dispatched through the GitHub Copilot SDK; \texttt{ask}/\texttt{say} read from and write to a configurable channel. \texttt{eval} deserializes a JSON-serialized workflow produced by an earlier \texttt{gen} and re-enters the interpreter in the caller's environment. Before each \texttt{gen}/\texttt{agent} call the interpreter constructs an authorization triplet \emph{(principal, action, resource)} and passes it to Cedar; the principal comes from the call-site's \texttt{as} clause (resolved through the environment if a variable), and a deny decision halts the workflow before the LLM call is made. The staged re-checking and policy-install invariants of \sect{sec:design-gradual} are implemented directly by the interpreter: a \texttt{Schema}-annotated binding re-invokes the type checker over the remaining expressions once the LLM output is resolved; an \texttt{enforce} expression runs the install invariants, merges the policy into a central store seeded by the \texttt{@policy} pragma, and compiles the result into a Cedar policy set. The full source is in the submission artifact.
