\section{Implementation}
\label{sec:implementation}

HADL is implemented in Rust. Source programs are parsed with a PEG grammar, and the AST is executed by a tree-walking interpreter with no code generation or intermediate representation.

\subsection{Runtime Execution Model}
\label{sec:impl-runtime}

The interpreter walks the AST and evaluates each expression in order. When it reaches a \texttt{Schema}-annotated binding, such as \texttt{crf} on line~5 of Figure~\ref{fig:clinical-trial}, it resolves the LLM's output to a concrete JS type and re-invokes the type checker over the remaining expressions so that a downstream use like \texttt{visit: crf} on line~13 is validated against the resolved shape. When it reaches an \texttt{enforce} expression, such as line~8, it runs the install invariants from Section~\ref{sec:design-gradual-policies}, compiles the merged policy into a Cedar policy set, and from that point on every \texttt{gen} and \texttt{agent} call is authorized before the LLM is invoked. If either check rejects the LLM output and \texttt{@retries} is active, provenance tracking walks back to the causal gen call and re-invokes it with the error as feedback.

Each primitive delegates to a dedicated backend. \texttt{js} expressions and backtick interpolation run on an embedded JavaScript engine (Boa) with HADL variables mirrored as globals. \texttt{gen} and \texttt{agent} calls are dispatched through the GitHub Copilot SDK. \texttt{ask} and \texttt{say} read from and write to the human operator via a configurable channel. The \texttt{eval} expression deserializes a JSON-serialized workflow produced by an earlier \texttt{gen} call and re-enters the interpreter in the caller's environment.

\subsection{Authorization Integration}
\label{sec:impl-auth}

Before each \texttt{gen} and \texttt{agent} call, the interpreter constructs an authorization triplet \emph{(principal, action, resource)} and passes it to Cedar's authorizer. The principal comes from the call-site's \texttt{as} clause, resolved through the environment if it is a variable reference and defaulting to \texttt{"gen"} or \texttt{"agent"} if absent. A deny decision halts the workflow before the LLM call is made. A policy file named by the \texttt{@policy} pragma seeds a central policy store at startup, and every subsequent \texttt{enforce} expression appends its rules to that same store. On Figure~\ref{fig:clinical-trial}, the \texttt{gen} call on line~13 runs under principal \texttt{role} and the policy permits it because generating a CRF visit is not an unblinding operation. A \texttt{coordinator} principal call requesting unblinding data would be denied.
