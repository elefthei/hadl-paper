\section{Conclusion}
\label{sec:conclusion}

We have presented HADL, a typed imperative domain-specific language for composing long-running agentic workflows. HADL treats agentic plans as programs, providing four domain-specific primitives (\texttt{ask}, \texttt{say}, \texttt{tool}, \texttt{agent}) as first-class typed expressions within an imperative language with mutable state, unbounded loops, structured iteration, and nested workflow composition. The central contribution is \emph{gradual generation}, a mechanism in which LLMs produce types, workflows, and authorization policies at runtime. \texttt{Schema} is universally compatible during static checking and triggers staged re-typechecking when concrete types materialize. Arrow types enable LLM-generated workflows executed via \texttt{eval}. The \texttt{Policy} type enables LLM-generated Cedar policies activated via \texttt{enforce}, with safety checks that reject deny-all rules and invalid action types, scoped principal validation, and append-only merging that can only tighten access. All three share provenance-driven self-healing. Cedar-based authorization is embedded directly in the evaluation semantics, enforcing (principal, action, resource) triplets before every tool and agent invocation.

Compile-time type checking catches structural errors before any LLM call is made. Gradual generation handles the dynamic nature of LLM outputs without sacrificing type safety where it can be statically enforced. The append-only semantics of policy merging, combined with Cedar's guarantee that \texttt{forbid} rules take precedence, ensures that dynamically generated policies can only tighten access.

Several directions remain for future work. Extending the language with parallel tool call expressions would enable fan-out patterns common in multi-agent workflows. Extending the type language with record and array types would bring field-level access under static checking. Supporting workflow imports would help scale to larger systems. A formal operational semantics with soundness proofs for the staged type checking mechanism would strengthen the theoretical foundation.
