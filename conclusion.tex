\section{Conclusion}
\label{sec:conclusion}

We have presented HADL, a typed imperative domain-specific language for agentic workflows. HADL treats agentic plans as programs. Its central contribution is \emph{gradual validation}, under which types, workflows, and authorization policies produced by the LLM at runtime are checked under the same type discipline that governs the rest of the program. \texttt{Schema} is universally compatible during static checking and triggers staged re-typechecking when concrete types materialize, workflow types enable LLM-generated workflows executed via \texttt{eval}, and the \texttt{Policy} type enables LLM-generated Cedar policies activated via \texttt{enforce} with a deny-all guard, scoped principal validation, and append-only merging that can only tighten access. All three share provenance-driven self-healing. Cedar-based authorization is embedded directly in the evaluation semantics, enforcing a (principal, action, resource) triplet before every \texttt{gen} and \texttt{agent} call, so the same type and \texttt{gen} information that catches errors also guides the LLM's generation, making safety and generation quality reinforce each other.

Several directions remain for future work. Clauses (T1)--(T3) of Theorem~\ref{thm:hadl-sound} are mechanized in Lean~4 under \texttt{mechanization/} in the artifact, with (T4) oracle-relative safety left as the remaining mechanization target. Extending the language with parallel gen call expressions would enable fan-out patterns common in multi-agent workflows. Supporting workflow imports would help scale to larger systems.
