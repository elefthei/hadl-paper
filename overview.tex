\section{Overview}
\label{sec:overview}

We introduce \hadl through two self-contained workflows. The clinical-trial workflow of Figure~\ref{fig:clinical-trial} covers gradual validation of types and policies. A second example covers workflow generation.

\hadl programs consist of workflow declarations, typed functions that compose four domain-specific primitives with standard imperative constructs. \texttt{ask} prompts a human operator and returns a \texttt{String}. \texttt{say} prints a string and returns unit. \texttt{gen} sends a description to an LLM gen session configured to return structured data and requires an explicit type annotation at the binding site. \texttt{agent} sends an instruction to an LLM reasoning session. These are first-class expressions in the language, composing with \texttt{let}/\texttt{var} bindings, \texttt{if}, \texttt{while}, \texttt{for}, and inline JavaScript (\texttt{js}).

\paragraph{Types and policies (Figure~\ref{fig:clinical-trial}).} Line~5 binds \texttt{crf} with a \texttt{Schema} annotation: when the LLM delivers the schema the interpreter resolves it and re-checks the remainder of the program against the concrete shape. If any field accessed later (such as \texttt{visit.cost} or \texttt{visit.patient\_id} in the \texttt{for} loop) is absent from the generated schema, the runtime retries the causal \texttt{gen} with the violated constraint as feedback before any visit data is collected. Lines~6--7 produce a blinding policy that line~8 activates via \texttt{enforce}, after which every \texttt{gen}/\texttt{agent} is authorized against the installed policy. The \texttt{for} loop on line~11 iterates patient IDs. Inside, \texttt{visit} is bound with the \texttt{crf} type annotation so every LLM output is validated against the concrete schema before the field accesses on lines~14--15 execute. Each \texttt{gen}/\texttt{agent} call carries an explicit \texttt{as} clause naming the principal. \texttt{@retries 3} on line~1 activates the feedback-driven self-healing budget (\sect{sec:design-gradual}): on any validation failure the runtime re-invokes the causal \texttt{gen} with the violated constraint appended to the original prompt.

\paragraph{Workflow generation.} Agentic workflows often delegate subtasks whose structure is itself the subject of the current reasoning step. \hadl makes workflows first-class: a \texttt{gen} annotated with a workflow type (e.g., a scorer \texttt{(car\_schema -> Number)} whose parameter type is a runtime-generated schema) instructs the LLM to produce a complete \hadl workflow as a JSON-serialized AST, and \texttt{eval} deserializes and invokes it in the caller's environment. At the \texttt{eval} site the type checker verifies argument and return types against the workflow type. An example (Figure~\ref{fig:overview-workflow}, in \sect{app:overview-example}) shows a car-recommendation outer workflow and the LLM-generated scorer that rates a car's comfort, cost, and safety and averages the three via \texttt{js}.
