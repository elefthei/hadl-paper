\section{Full Reduction Rules}
\label{app:rules}

This appendix lists the reduction rules elided from \sect{sec:sem-reduction} --- the remaining pure-core root rules, the ask/say rules, closure application, and the structural congruence rules --- followed by the proofs of Theorems~\ref{thm:hadl-soundness}, \ref{thm:hadl-progress}, and \ref{thm:hadl-term}. Left-to-right call-by-value evaluation order is enforced throughout by the structural congruence rules. The substitution $\Inst{e}{v}$ denotes the capture-avoiding instantiation of de-Bruijn index $0$ in $e$ with the value $v$ (decrementing all other free indices); for a closure body with $n$ binders, simultaneous substitution $e[v_0/0,\dots,v_{n-1}/(n{-}1)]$ is used.

\subsection{Pure Core (L1)}
\label{app:rules-core}

\begin{mathpar}
\inferrule*[right=\rulename{If-True}]
  { }
  {\MConfig{\MErr}{\MPol}{\Iff{\texttt{true}}{e_1}{e_2}} \step \MConfig{\MErr}{\MPol}{e_1}}

\inferrule*[right=\rulename{If-False}]
  { }
  {\MConfig{\MErr}{\MPol}{\Iff{\texttt{false}}{e_1}{e_2}} \step \MConfig{\MErr}{\MPol}{e_2}}

\inferrule*[right=\rulename{While}]
  { }
  {\MConfig{\MErr}{\MPol}{\Whl{e}{e'}} \step \MConfig{\MErr}{\MPol}{\Iff{e}{\Seq{e'}{\Whl{e}{e'}}}{\Tunit}}}

\inferrule*[right=\rulename{For-Nil}]
  { }
  {\MConfig{\MErr}{\MPol}{\Forr{[\,]}{e}} \step \MConfig{\MErr}{\MPol}{\Tunit}}

\inferrule*[right=\rulename{For-Cons}]
  {\mathsf{isValue}(v)}
  {\MConfig{\MErr}{\MPol}{\Forr{v :: vs}{e}} \step \MConfig{\MErr}{\MPol}{\Seq{\Inst{e}{v}}{\Forr{vs}{e}}}}

\inferrule*[right=\rulename{Seq}]
  {\mathsf{isValue}(v)}
  {\MConfig{\MErr}{\MPol}{\Seq{v}{e}} \step \MConfig{\MErr}{\MPol}{e}}

\inferrule*[right=\rulename{Js}]
  {\mathsf{jsEval}(c) = v \\ \mathsf{isValue}(v)}
  {\MConfig{\MErr}{\MPol}{\Js{c}} \step \MConfig{\MErr}{\MPol}{v}}
\end{mathpar}

\subsection{Ask, Say, and Closure Application}
\label{app:rules-io}

\begin{mathpar}
\inferrule*[right=\rulename{Ask}]
  {\Oracle{s}{\MErr}{\Tstring} \ni v \\
   \mathsf{isValue}(v) \\
   \RtType{v}{\Tstring}}
  {\MConfig{\MErr}{\MPol}{\Ask{s}} \step \MConfig{\MErr \cdot \SucEv}{\MPol}{v}}

\inferrule*[right=\rulename{Say}]
  { }
  {\MConfig{\MErr}{\MPol}{\Say{s}} \step \MConfig{\MErr}{\MPol}{\Tunit}}

\inferrule*[right=\rulename{Eval-Success}]
  {|\overline{v_i}| = n \\ \forall i.\ \mathsf{isValue}(v_i)}
  {\MConfig{\MErr}{\MPol}{\Eval{(\Clos{n}{e_b})}{\overline{v_i}}} \step \MConfig{\MErr}{\MPol}{e_b[v_0/0,\dots,v_{n-1}/(n{-}1)]}}
\end{mathpar}

$\rulename{Eval-Success}$ applies a closure $\Clos{n}{e_b}$ to a list of $n$ values by simultaneous substitution. Unlike the environment-based formulation, there is no environment extension and no per-argument $\RtType{\cdot}{\cdot}$ side condition at the application site: each $v_i$ was produced by an earlier reduction step whose own rule carried the runtime-type obligation (T2 then guarantees every value has \emph{some} runtime type; the black-boxed static checker handles shape agreement). Lifting $\rulename{Eval-Success}$ to non-root positions is standard; the function position of $\Eval{f}{\overline{v_i}}$ is driven to a closure value by $\rulename{Eval-Fun-Cong}$ below.

\subsection{Structural Congruence}
\label{app:rules-cong}

\begin{mathpar}
\inferrule*[right=\rulename{If-Cong}]
  {\MConfig{\MErr}{\MPol}{e_1} \step \MConfig{\MErr'}{\MPol'}{e_1'}}
  {\MConfig{\MErr}{\MPol}{\Iff{e_1}{e_2}{e_3}} \step \MConfig{\MErr'}{\MPol'}{\Iff{e_1'}{e_2}{e_3}}}

\inferrule*[right=\rulename{Seq-Cong}]
  {\MConfig{\MErr}{\MPol}{e_1} \step \MConfig{\MErr'}{\MPol'}{e_1'}}
  {\MConfig{\MErr}{\MPol}{\Seq{e_1}{e_2}} \step \MConfig{\MErr'}{\MPol'}{\Seq{e_1'}{e_2}}}

\inferrule*[right=\rulename{For-Cong}]
  {\MConfig{\MErr}{\MPol}{e_1} \step \MConfig{\MErr'}{\MPol'}{e_1'}}
  {\MConfig{\MErr}{\MPol}{\Forr{e_1}{e_2}} \step \MConfig{\MErr'}{\MPol'}{\Forr{e_1'}{e_2}}}

\inferrule*[right=\rulename{Enforce-Cong}]
  {\MConfig{\MErr}{\MPol}{e} \step \MConfig{\MErr'}{\MPol'}{e'}}
  {\MConfig{\MErr}{\MPol}{\Enforce{e}} \step \MConfig{\MErr'}{\MPol'}{\Enforce{e'}}}

\inferrule*[right=\rulename{Eval-Fun-Cong}]
  {\MConfig{\MErr}{\MPol}{f} \step \MConfig{\MErr'}{\MPol'}{f'}}
  {\MConfig{\MErr}{\MPol}{\Eval{f}{\overline{v_i}}} \step \MConfig{\MErr'}{\MPol'}{\Eval{f'}{\overline{v_i}}}}
\end{mathpar}

None of the congruence rules carries a $\neq \mathsf{err}$ side condition: the language has no $\mathsf{errTerm}$ constructor, so a stuck configuration has no outgoing transition and its stuckness simply propagates up by absence.

There is no separate heal rule for $\Enforce{\cdot}$, for $\Agent{\cdot}{\cdot}$, or for closure application: the unified $\rulename{Oracle-Heal-Type}$ and $\rulename{Oracle-Heal-Pol}$ rules in \figr{fig:rules-core} subsume all gen/agent self-healing via the $\mathtt{OAction}$ action tag, and an $\Enforce{\cdot}$ whose policy value fails to install simply gets stuck --- there is no rewind-to-gen fallback in the mechanized semantics. Earlier drafts of the formalization included an $\rulename{Enforce-Heal}$ rule; it has been removed together with the $\mathsf{errTerm}$ constructor that it targeted.

\section{Proofs of Theorems~\ref{thm:hadl-soundness}--\ref{thm:hadl-term}}
\label{app:proofs}

Each proof is an induction on the derivation of $C \step C'$, mirroring the Lean case split on the constructor of $\mathtt{Step}$. The mechanization is free of user-introduced axioms: $\mathtt{policyInstall\_shrinks}$ is proved by iterated application of Cedar-Lean's $\mathtt{unchanged\_deny\_when\_add\_forbid}$ theorem, and the JavaScript host evaluator is modelled by an opaque $\mathtt{jsEval} : \mathtt{JsExpr} \to \mathsf{Option}\;\mathtt{Expr}$. All theorems depend only on Lean's standard meta-axioms ($\mathtt{propext}$, $\mathtt{Classical.choice}$, $\mathtt{Quot.sound}$); no other steps are admitted.

\subsection{T1: WF-preservation}
\label{app:proof-sound-a}

The invariant is the single clause $\retries{\MErr} \le \MRetry$. The proof is by cases on the $\mathtt{Step}$ constructor in \texttt{Soundness.lean}; the non-trivial cases are:

\paragraph{Case $\rulename{Ask}$, $\rulename{Oracle-Success}$.} The target heal context is $\MErr \cdot \SucEv$; by the trailing-success lemma $\mathtt{ErrCtx.retries\_append\_success}$ its retry metric is $0 \le \MRetry$.

\paragraph{Case $\rulename{Oracle-Heal-Type}$, $\rulename{Oracle-Heal-Pol}$.} The rule's own premise $\retries{\MErr \cdot \ErrEv{\epsilon}} \le \MRetry$ is exactly the required invariant on $C'$.

\paragraph{Pure-core and congruence cases.} $\MErr$ is either unchanged or inherits $\le \MRetry$ from the inductive hypothesis on the inner step.

Because $\WF{C}$ no longer involves environment well-typedness or residual static typeability, the per-rule cases are uniformly trivial --- this is the mechanized content of $\mathtt{T1\_WF\_preservation}$.

\subsection{T2: Staged materialization}
\label{app:proof-sound-b}

If the step $C \step C'$ produces a value $v$ at $C'$'s expression slot, then $\exists \tau.\ \RtType{v}{\tau}$. The proof ($\mathtt{T2\_staged\_materialization}$) factors through a value-lemma ``$\mathsf{isValue}(v) \Rightarrow \exists \tau.\ \RtType{v}{\tau}$'' that enumerates the value constructors. For base literals and schema/policy values the corresponding $\mathtt{vUnit}/\mathtt{vBool}/\mathtt{vInt}/\mathtt{vStr}/\mathtt{vSchema}/\mathtt{vPol}$ case applies. For closures, the $\mathtt{vClos}$ case assigns $\tau = \Tarrow{\overline{\Tunit}}{\Tunit}$ of matching arity --- the precise argument and return types of the body are irrelevant to T2's existential claim. For record and array values, $\mathtt{vRec}$ and $\mathtt{vArr}$ assign $\Tunit$-valued record/array types, again as existential witnesses. This is strictly stronger than the prior ``materialization correctness'' claim, which only covered values returned by $\rulename{Gen-Success}$ at the annotated type; the new statement applies uniformly at every reduced expression and removes the need for the staged-materialization sub-argument in downstream proofs.

\subsection{T3: Policy monotonicity}
\label{app:proof-sound-c}

$\MPol$ is updated only by $\rulename{Enforce-Install}$, whose target policy $\MPol' = \Install{\MPol}{p}$ satisfies $\Allow{\MPol'} \subseteq \Allow{\MPol}$ by $\mathtt{policyInstall\_shrinks}$. All other rules leave $\MPol$ unchanged and give $\Allow{\MPol'} = \Allow{\MPol}$, hence the reflexive subset. The pointwise authorization claim is immediate because $\rulename{Oracle-Success}$, $\rulename{Oracle-Heal-Type}$ all carry $\PolOk{\MPol}{a.\MPrin}{a.\mathsf{eff}}$ as a side condition (and $\rulename{Oracle-Heal-Pol}$ carries its negation, which is why it exists).

\subsection{T4a: Budget → no heal}
\label{app:proof-prog-a}

Assume $C = \MConfig{\MErr}{\MPol}{a.e}$ with $\retries{\MErr} > \MRetry$ and $C \step C'$. We case-split on the $\mathtt{Step}$ constructor producing $C'$. Every pure-core, congruence, and $\mathtt{enforceInstall}/\mathtt{evalSuccess}/\mathtt{letBind}/\ldots$ case immediately contradicts $e = a.e$ by constructor distinctness. The $\rulename{Oracle-Heal-Type}$ and $\rulename{Oracle-Heal-Pol}$ cases have $\retries{\MErr \cdot \ErrEv{\epsilon}} \le \MRetry$ as a premise; rewriting with $\mathtt{ErrCtx.retries\_append\_error}$ yields $\retries{\MErr} + 1 \le \MRetry$, contradicting the hypothesis $\retries{\MErr} > \MRetry$ by $\mathtt{omega}$. The only surviving case is $\rulename{Oracle-Success}$, which indeed transitions to $\MConfig{\MErr \cdot \SucEv}{\MPol}{v}$. This is mechanized as $\mathtt{T4\_budget\_no\_heal}$ in \texttt{Safety.lean}.

Notably, T4a does not claim that $C$ is \emph{stuck} --- an oracle that is still truthful at the exhausted heal context can still fire $\rulename{Oracle-Success}$. The claim is only that no \emph{heal} rule applies, which is the semantically meaningful part: a budget-exhausted configuration cannot accumulate more feedback events.

\subsection{T4b: Truthful success}
\label{app:proof-prog-b}

Fix an oracle redex $a.e$ (either $\Gen{\tau}{s}{\MPrin}$ or $\Agent{s}{\MPrin}$) with $\PolOk{\MPol}{\MPrin}{a.\mathsf{eff}}$. By \defn{def:oracle-truthful} there exists some $\MErr'$ with $\retries{\MErr'} \le \MRetry$ and a value $v$ with $\Oracle{s}{\MErr'}{\tau} \ni v$, $\mathsf{isValue}(v)$, and $\RtType{v}{\tau}$. Apply $\rulename{Oracle-Success}$ at the action tag $a$: all four side conditions are discharged, so $\MConfig{\MErr'}{\MPol}{a.e} \step \MConfig{\MErr' \cdot \SucEv}{\MPol}{v}$. Mechanized as $\mathtt{T4\_truthful\_success}$ (the agent variant is $\mathtt{T4\_truthful\_success\_agent}$; it uses a direct oracle witness at the caller's $\MErr$ because the $\mathtt{eventuallyTruthful}$ predicate returns only an unspecified heal context).

\subsection{T4c: Progress-gen, Progress-agent}
\label{app:proof-prog-c}

These are wrappers around T4b: they add $\PolOk{\MPol}{\MPrin}{a.\mathsf{eff}}$ as an explicit hypothesis and take a direct oracle witness $\exists v.\ \mathsf{isValue}(v) \wedge \RtType{v}{\tau} \wedge \Oracle{s}{\MErr}{\tau} \ni v$ at the caller's \emph{current} heal context. The conclusion $\exists C'.\ C \step C'$ follows by applying $\rulename{Oracle-Success}$ exactly as in T4b but without the intermediate ``reach some $\MErr'$'' step. Mechanized as $\mathtt{T4\_progress\_gen}$ / $\mathtt{T4\_progress\_agent}$ in \texttt{Safety2.lean}.

\subsection{Termination}
\label{app:proof-term}

This argument is on paper. Between two successive oracle firings, only pure-core and congruence rules fire. Pure-core reduction terminates on any expression whose $\texttt{while}$ and $\texttt{for}$ loops terminate (assumed of $e_0$); substitution is finitary and preserves the termination measure on loop-free code. At each oracle firing either $\SucEv$ is appended (resetting $\retries{\cdot}$ to $0$ by T1) or an $\ErrEv{\cdot}$ extends the trailing retry run by one. After at most $\MRetry + 1$ heals the configuration is either pushed into the ``budget exceeded'' regime of T4a --- where only $\rulename{Oracle-Success}$ can fire, so the configuration is either truthful-success-progressing or stuck by absence of transition --- or has already succeeded. Under eventually-truthful $\mathcal{O}$, truthful-success is guaranteed at every site; since the number of oracle sites reached along the trace is finite (bounded by $|e_0|$ plus finitely many $\rulename{Eval-Success}$ expansions), the trace terminates in a value configuration. Otherwise the trace either terminates in a value or gets stuck at an oracle redex for which no witness is ever produced; no ill-formed configuration, and no $\mathsf{errTerm}$ marker, is ever reached, since neither exists in the language.
